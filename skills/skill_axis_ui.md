# AXIS - UI Principles

## Design Philosophy
Hilux por fora, Porsche por dentro. Simple interface that solves problems, robust engine underneath.

## Core Goals
Minimal friction, session-first workflow, fast data entry, structured documentation, immediate value.

## Onboarding Light (ABA)
Zero wizard, zero gamification, value in less than 5 minutes, two screens max. Button 1: Personalizar sua Clinica. Button 2: Cadastrar Primeiro Aprendiz.

## Soft Validation (Not Hard Stop)
Level 1: Yellow alert, can save. Level 2: Orange + explanation, can save. Level 3-4: Requires justification. Level 5+: Requires Supervisor approval.

## Dashboard Priority
Main button: Gerar Relatorio para Convenio (not Novo Protocolo). Entry point is immediate problem.

## Avoid
Complex dashboards, excessive charts, unnecessary clicks, gamification, long wizards, mandatory videos.

## Preferred Flows
TCC: Patient -> Session -> Events -> Processing -> Report
ABA: Learner -> Protocol -> Session -> Trials/Behaviors -> Snapshot -> Report

## Tooltips
Every field has ? icon with definition, example, and Bible reference. Tooltips replace training videos.
